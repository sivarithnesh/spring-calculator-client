We have completed the review and correction of the reports that were previously reported as returning blank results or incorrect data.

Root Cause Analysis

After detailed investigation, we identified that the primary issue was related to case-sensitive string comparisons in Oracle SQL queries.

Several report datasets contained filter conditions where the values specified in the SQL did not exactly match the case of the corresponding values stored in the database.

  • TP-Sett vs TP-SETT
  • Monthly vs MONTHLY
  • Resolved vs RESOLVED

Since Oracle performs exact string comparisons, these mismatches caused the queries to return zero records, resulting in reports appearing blank even though valid data existed in the database.

Fix Applied

We reviewed and updated the affected datasets by standardizing string comparisons and implementing appropriate case handling using UPPER() functions where required.

Validation Performed

  • Reviewed all affected report datasets
  • Corrected case-sensitive filter conditions
  • Validated the updated SQL queries using production-cut data
  • Verified that the reports are now returning expected results
  • Reviewed associated tabs, sub-datasets, and supporting queries
